# План публикации сайта на домен plaxotin.ru (Reg.ru)

## Обзор проекта

Ваш сайт — это React/TypeScript портфолио, собранное с помощью Vite. При сборке (`npm run build`) создаётся статическая папка `dist/` с готовыми HTML/CSS/JS файлами, которые можно разместить на любом хостинге.

---

## Пошаговый план развёртывания

### Этап 1: Подготовка файлов сайта (на вашем компьютере)

#### 1.1. Клонирование репозитория (если ещё не сделано)

```bash
git clone https://github.com/Plaxotin/konstantin-portfolio.git
cd konstantin-portfolio
```

#### 1.2. Установка зависимостей

```bash
npm install
```

#### 1.3. Сборка production-версии

```bash
npm run build
```

После выполнения команды появится папка `dist/` со всеми файлами сайта.

#### 1.4. Создание ZIP-архива

Создайте ZIP-архив из содержимого папки `dist/`:

**Windows:**
- Откройте папку `dist/`
- Выделите все файлы внутри (Ctrl+A)
- ПКМ → «Отправить» → «Сжатая ZIP-папка»
- Назовите архив `site.zip`

**Mac/Linux:**
```bash
cd dist
zip -r ../site.zip .
cd ..
```

> ⚠️ **Важно:** архивируйте содержимое папки `dist/`, а не саму папку. Файл `index.html` должен быть в корне архива, а не внутри подпапки.

### Этап 2: Привязка домена к хостингу Reg.ru

#### 2.1. Вход в личный кабинет

1. Перейдите на [reg.ru](https://www.reg.ru/)
2. Нажмите «Вход» → введите логин и пароль
3. Перейдите в [личный кабинет](https://www.reg.ru/user/account/#/)

#### 2.2. Привязка домена plaxotin.ru к хостингу

**Автоматический способ (рекомендуется):**

1. В личном кабинете выберите вашу **услугу хостинга**
2. Нажмите кнопку **«Добавить сайт»**
3. Во вкладке «На аккаунте» выберите домен **plaxotin.ru**
4. Включите переключатель **«Автоматически настроить DNS-серверы и записи зоны»**
5. Нажмите **«Подключить»**

**Ручной способ (если автоматический не сработал):**

**Шаг A: Измените DNS-серверы домена**
1. Перейдите в раздел «Домены» → выберите **plaxotin.ru**
2. Найдите блок «DNS-серверы и управление зоной» → нажмите **«Изменить»**
3. Выберите DNS-серверы:
   - `ns1.hosting.reg.ru`
   - `ns2.hosting.reg.ru`
4. Подтвердите изменения

**Шаг B: Добавьте домен в панель управления хостингом**
1. Откройте панель управления хостингом (ISPManager)
2. Перейдите в раздел **«Сайты»**
3. Нажмите **«Создать сайт»**
4. Введите домен: `plaxotin.ru`
5. Нажмите **«ОК»**

> ⏱️ **Обновление DNS занимает до 24 часов.** В этот период сайт может быть недоступен по домену.

---

### Этап 3: Загрузка файлов сайта на хостинг

#### 3.1. Вход в панель управления ISPManager

1. В личном кабинете Reg.ru перейдите в раздел **«Услуги хостинга»**
2. Нажмите на иконку монитора (🖥️) рядом с вашей услугой — откроется **ISPManager**

#### 3.2. Переход к файлам сайта

1. В ISPManager перейдите в раздел **«Сайты»**
2. Выберите домен **plaxotin.ru**
3. Нажмите **«Файлы сайта»**

#### 3.3. Очистка папки сайта

1. Удалите все файлы в папке, **кроме директории `webstat`** (если она есть)
2. Это файлы парковочной страницы Reg.ru — они вам не нужны

#### 3.4. Загрузка архива

1. Нажмите кнопку **«Загрузить»**
2. Выберите созданный ранее архив `site.zip`
3. Дождитесь завершения загрузки

#### 3.5. Распаковка архива

1. Выделите загруженный архив `site.zip`
2. Нажмите **«Извлечь»**
3. Убедитесь, что файлы извлеклись прямо в корневую папку сайта

#### 3.6. Проверка структуры файлов

После распаковки в корневой папке должны быть:
```
/www/plaxotin.ru/
├── index.html          ← главный файл
├── assets/             ← папка с JS/CSS
│   ├── index-XXXXX.js
│   └── index-XXXXX.css
├── 404.html            ← страница ошибки
└── (другие файлы)
```

> ⚠️ Если файлы оказались во вложенной папке (например, `/www/plaxotin.ru/dist/`), переместите их в корень с помощью кнопок **«Вырезать»** и **«Вставить»**.

---

### Этап 4: Проверка работоспособности

#### 4.1. Проверка до обновления DNS (необязательно)

Если DNS ещё не обновились, можно проверить сайт через техническое доменное имя:
- В ISPManager → «Сайты» → найдите технический домен вида `u1234567.plsk.regruhosting.ru`
- Откройте его в браузере

#### 4.2. Проверка основного домена

После обновления DNS (до 24 часов) откройте в браузере:
- **http://plaxotin.ru**
- **http://www.plaxotin.ru**

Сайт должен отображаться корректно.

---

### Этап 5: Настройка SSL-сертификата (HTTPS)

Для безопасного соединения установите бесплатный SSL-сертификат Let's Encrypt:

1. В ISPManager перейдите в раздел **«SSL-сертификаты»**
2. Нажмите **«Let's Encrypt»**
3. Выберите домен **plaxotin.ru**
4. Включите опцию **«Перенаправлять HTTP на HTTPS»**
5. Нажмите **«Выпустить»**

После выпуска сертификата сайт будет доступен по адресу:
- **https://plaxotin.ru** ✅

## Альтернативные способы загрузки файлов

### Через FTP

1. Скачайте FTP-клиент (например, [FileZilla](https://filezilla-project.org/))
2. Данные для подключения находятся в письме от Reg.ru:
   - **Хост:** ftp.u1234567.plsk.regruhosting.ru (уточните в письме)
   - **Логин:** u1234567 (ваш логин хостинга)
   - **Пароль:** (указан в письме)
   - **Порт:** 21
3. Подключитесь и загрузите файлы в папку `/www/plaxotin.ru/`

### Через SSH (для продвинутых пользователей)

```bash
# Подключение по SSH
ssh u1234567@u1234567.ssh.regruhosting.ru

# Переход в папку сайта
cd www/plaxotin.ru

# Загрузка файлов через SCP (с локального компьютера)
scp -r dist/* u1234567@u1234567.ssh.regruhosting.ru:www/plaxotin.ru/
```

## Обновление сайта в будущем

При необходимости обновить сайт:

1. Внесите изменения в код
2. Выполните `npm run build`
3. Создайте новый ZIP-архив из папки `dist/`
4. Загрузите на хостинг через ISPManager (или FTP/SSH)
5. Замените старые файлы новыми
